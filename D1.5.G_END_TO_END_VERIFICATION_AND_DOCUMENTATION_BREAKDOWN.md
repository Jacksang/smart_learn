### Exact end-to-end loop D1.5.G must prove
1. Start from an owned project with persisted questions mapped to outline items.
2. Submit one or more answers through `POST /api/projects/:projectId/questions/:questionId/answers`.
3. Verify the persisted attempts are retrievable through:
   - `GET /api/projects/:projectId/questions/:questionId/answers`
   - `GET /api/projects/:projectId/answers/history`
4. Re-evaluate a persisted attempt through `POST /api/projects/:projectId/answers/evaluate` using `answerAttemptId`, proving evaluation reads stored attempt + question context instead of requiring resubmission.
5. Refresh progress through `POST /api/projects/:projectId/progress/refresh`, proving answer-attempt data drives persisted `progress_snapshots`.
6. Read back the persisted progress surfaces through:
   - `GET /api/projects/:projectId/progress`
   - `GET /api/projects/:projectId/progress/topics/:itemId`
   - `GET /api/projects/:projectId/progress/weak-areas`
7. Assert the final retrieval payloads are consistent with the submitted attempts, deterministic evaluation rules, and persisted topic/project snapshot data.

### Minimum documentation/closeout contract for D1.5.G
- Integration evidence must cover the whole persisted D1.5 loop, split as:
  - G2: submission + question/project answer history
  - G3: explicit evaluation by stored `answerAttemptId`
  - G4: progress refresh + project/topic/weak-area retrieval from persisted snapshots
- Documentation closeout must include one concise answers/progress flow doc describing:
  - the endpoint sequence above
  - that `answer_attempts` is the persisted source for attempts/evaluation context
  - that `progress_snapshots` is the persisted source for project/topic/weak-area retrieval
  - MVP limits: deterministic scoring and heuristic weak-area classification
- Final checklist closeout must update D1.5 tracking only after focused D1.5 tests and new integration coverage pass together.

- Focused verification passed in:

  - `backend/src/answers/repository.test.js`
- G3 is now unblocked and ready to resume with the explicit-evaluation integration test slice.
