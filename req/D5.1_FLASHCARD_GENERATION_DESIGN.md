# D5.1 Flashcard Generation - Design Document

## Purpose
This document describes the design for Smart Learn's Flashcard Generation system - an automated review system that creates effective flashcards from student study materials using intelligent concept extraction and spaced repetition scheduling.

## Overview
The Flashcard Generation system transforms uploaded study materials into personalized review flashcards that:

1. **Extract** key concepts and definitions from study content
2. **Generate** questions at appropriate cognitive levels
3. **Schedule** reviews using proven spaced repetition algorithms
4. **Adapt** to student performance for optimal learning

This creates an automated, effective review system that saves students time while maximizing retention.

## Core Principles

### 1. Content-Driven Generation
- Flashcards are derived directly from student's own study materials
- No external content required
- Maintains learning context and relevance
- Supports multiple file formats (PDF, DOCX, images, text)

### 2. Cognitive Level Diversity
- Questions at recall, understanding, and application levels
- Balance between memorization and comprehension
- Progressive difficulty based on student performance
- Multiple question types for varied review experiences

### 3. Spaced Repetition Optimization
- SM-2 algorithm variant for proven effectiveness
- Individual scheduling per flashcard
- Performance-based interval adjustment
- Prevents over-review or under-review

### 4. Adaptive Learning
- System learns from student performance
- Adjusts difficulty and scheduling automatically
- Tracks review history and patterns
- Improves future flashcard generation

## Component 1: Concept Extraction

### Extraction Algorithms

**1. Keyword Extraction**
- Noun phrase identification
- Technical term recognition
- Frequency-based prioritization
- Context-aware filtering

**2. Definition Pattern Detection**
- "X is defined as..." patterns
- "X refers to..." patterns
- Mathematical formula recognition
- Scientific notation handling

**3. Relationship Mapping**
- Causal relationships (A causes B)
- Hierarchical relationships (A is a type of B)
- Temporal relationships (A happens before B)
- Functional relationships (A enables B)

**4. Concept Prioritization**
- Frequency scoring (appears often = important)
- Position scoring (intro/conclusion = important)
- Relationship density (well-connected = important)
- Context relevance (matches learning goals = important)

**Algorithm:**
```javascript
function extractConcepts(content) {
  const concepts = [];
  
  // Tokenize content
  const tokens = tokenize(content);
  
  // Identify noun phrases and technical terms
  const candidates = identifyNounPhrases(tokens);
  
  // Extract definitions
  const definitions = extractDefinitions(candidates);
  
  // Calculate relationship networks
  const relationships = mapRelationships(definitions);
  
  // Prioritize concepts
  concepts = prioritizeConcepts(definitions, relationships);
  
  // Format output
  return {
    concepts: concepts.map(c => ({
      term: c.term,
      definition: c.definition,
      relatedTerms: c.relatedTerms,
      priority: calculatePriority(c),
      confidence: c.confidence
    })),
    metadata: {
      totalConcepts: concepts.length,
      processingTime: performance.now()
    }
  };
}
```

## Component 2: Question Generation

### Question Templates by Cognitive Level

**Level 1: Recall**
- "What is [term]?"
- "Define [term]"
- "What does [term] refer to?"

**Level 2: Understanding**
- "Explain [term] in your own words"
- "What is the significance of [term]?"
- "How does [term] relate to [related term]?"

**Level 3: Application**
- "How would you use [term] to solve [problem]?"
- "Apply [term] to a real-world scenario"
- "What would happen if [term] didn't exist?"

### Question Types

**1. Multiple Choice (MCQ)**
- 1 correct answer + 3 distractors
- Distractors generated from:
  - Related concepts (plausible alternatives)
  - Common misconceptions
  - Similar-sounding terms
- Randomized answer positions

**Example:**
```json
{
  "type": "multiple_choice",
  "question": "What is the primary function of photosynthesis?",
  "options": [
    "To generate energy for the plant",
    "To store energy as glucose",
    "To produce oxygen as a byproduct",
    "To convert carbon dioxide into water"
  ],
  "correctAnswer": 0,
  "cognitiveLevel": "recall",
  "term": "Photosynthesis"
}
```

**2. Fill-in-the-Blank**
- Remove key term from definition
- Student must provide exact term
- Case-insensitive matching
- Synonym recognition

**Example:**
```json
{
  "type": "fill_blank",
  "question": "The process by which plants convert light energy into chemical energy is called ______.",
  "answer": "photosynthesis",
  "cognitiveLevel": "recall",
  "term": "Photosynthesis"
}
```

**3. Open-Ended**
- Requires explanation or application
- Evaluated for key concepts present
- Accepts multiple valid responses
- Encourages deeper understanding

**Example:**
```json
{
  "type": "open_ended",
  "question": "Explain how photosynthesis and cellular respiration are interconnected processes.",
  "keyConcepts": ["oxygen", "carbon dioxide", "energy", "glucose"],
  "cognitiveLevel": "application",
  "term": "Photosynthesis"
}
```

### Question Quality Validation

**Checklist:**
- ✅ Question is clear and unambiguous
- ✅ Answer is definitive (except open-ended)
- ✅ Distractors are plausible but clearly wrong
- ✅ Cognitive level matches intended complexity
- ✅ Term is central to understanding
- ✅ No leading language or hints

**Validation Function:**
```javascript
function validateQuestion(question) {
  const errors = [];
  
  // Check clarity
  if (question.question.length < 10) {
    errors.push("Question too short");
  }
  
  // Check answer clarity
  if (question.correctAnswer === undefined && !question.answer) {
    errors.push("Missing correct answer");
  }
  
  // Check distractor quality
  if (question.options) {
    const uniqueOptions = new Set(question.options);
    if (uniqueOptions.size !== question.options.length) {
      errors.push("Duplicate options");
    }
  }
  
  // Check cognitive level
  if (!VALID_COGNITIVE_LEVELS.includes(question.cognitiveLevel)) {
    errors.push("Invalid cognitive level");
  }
  
  return {
    isValid: errors.length === 0,
    errors
  };
}
```

## Component 3: Spaced Repetition Algorithm

### SM-2 Algorithm Details

**Core Concept:**
- Each flashcard has an interval that increases with each successful review
- Interval resets if review fails
- Difficulty rating affects interval multiplier

**SM-2 Variables:**
- `interval`: Days until next review
- `repetitions`: Number of successful consecutive reviews
- `difficulty`: 0-5 scale (0=too hard, 5=easy)

**SM-2 Formulas:**
```javascript
function calculateNextReview(flashcard, rating) {
  if (rating < 3) {
    // Failed review
    return {
      interval: 1,
      repetitions: 0,
      nextReviewDate: new Date(Date.now() + 24 * 60 * 60 * 1000)
    };
  }
  
  // Successful review
  if (flashcard.repetitions === 0) {
    return {
      interval: 1,
      repetitions: 1,
      nextReviewDate: new Date(Date.now() + 24 * 60 * 60 * 1000)
    };
  } else if (flashcard.repetitions === 1) {
    return {
      interval: 6,
      repetitions: 2,
      nextReviewDate: new Date(Date.now() + 6 * 24 * 60 * 60 * 1000)
    };
  } else {
    // For repetitions >= 2
    const newInterval = Math.round(flashcard.interval * Math.exp((rating - 3) * 0.1));
    return {
      interval: Math.max(newInterval, flashcard.interval + 1),
      repetitions: flashcard.repetitions + 1,
      nextReviewDate: calculateNextReviewDate(flashcard.repetitions + 1)
    };
  }
}
```

**Schedule Management:**
- Track review history per flashcard
- Calculate next review date dynamically
- Group by review dates for batch scheduling
- Handle time zone conversions

**Review Scheduling:**
```javascript
function getReviewSchedule(studentId, targetDate) {
  const flashcards = getStudentFlashcards(studentId);
  const reviews = [];
  
  flashcards.forEach(card => {
    if (card.nextReviewDate <= targetDate && !card.isMastered) {
      reviews.push({
        flashcardId: card.id,
        dueDate: card.nextReviewDate,
        type: card.type,
        topic: card.topic
      });
    }
  });
  
  return {
    targetDate,
    totalDue: reviews.length,
    flashcards: reviews
  };
}
```

## Component 4: Service Layer Integration

### Flashcard Creation Workflow

**Steps:**
1. Parse student's uploaded materials
2. Extract key concepts (prioritized)
3. For each concept:
   - Generate recall-level question
   - Generate understanding-level question (optional)
   - Create spaced repetition schedule
4. Store flashcards in database
5. Return generated flashcards to user

**Batch Processing:**
- Process multiple files simultaneously
- Group flashcards by topic
- Generate different question types per concept
- Track generation progress

**Service Functions:**
```javascript
class FlashcardService {
  constructor() {
    this.conceptExtractor = new ConceptExtractor();
    this.questionGenerator = new QuestionGenerator();
    this.spacedRepetition = new SpacedRepetition();
  }
  
  async generateFlashcards(projectId, materials, options = {}) {
    const concepts = await this.conceptExtractor.extract(materials);
    const flashcards = [];
    
    // Prioritize by importance
    concepts.sort((a, b) => b.priority - a.priority);
    
    for (const concept of concepts.slice(0, options.maxFlashcards || 50)) {
      const questions = this.questionGenerator.generate(concept);
      
      questions.forEach(question => {
        const flashcard = {
          projectId,
          conceptTerm: concept.term,
          conceptDefinition: concept.definition,
          question,
          cognitiveLevel: question.cognitiveLevel,
          spacedRepetition: this.spacedRepetition.initialize(concept.priority)
        };
        
        flashcards.push(flashcard);
      });
    }
    
    return {
      flashcards,
      metadata: {
        totalGenerated: flashcards.length,
        sourceConcepts: concepts.length
      }
    };
  }
}
```

**New Tables:**
```sql
CREATE TABLE review_flashcards (
  id UUID PRIMARY KEY,
  project_id UUID REFERENCES learning_projects(id),
  concept_term VARCHAR(255),
  concept_definition TEXT,
  question_type VARCHAR(50),
  question_content JSONB,
  correct_answer JSONB,
  cognitive_level VARCHAR(50),
  topic VARCHAR(100),
  difficulty_initial INT DEFAULT 3,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_attempts (
  id UUID PRIMARY KEY,
  flashcard_id UUID REFERENCES review_flashcards(id),
  student_id UUID,
  rating INT CHECK (rating BETWEEN 0 AND 5),
  correct BOOLEAN,
  response_time_seconds INT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_schedules (
  flashcard_id UUID REFERENCES review_flashcards(id),
  next_review_date DATE,
  interval_days INT,
  repetitions INT,
  status VARCHAR(50) DEFAULT 'active',
  updated_at TIMESTAMP DEFAULT NOW(),
  PRIMARY KEY (flashcard_id)
);
```
