## Executive Summary

This requirement defines the implementation of a complete PostgreSQL database integration layer for Smart Learn, including voice interaction tracking. The implementation will provide persistent storage for all learning data, support complex queries for progress analytics, and track voice-based interactions.

## Database Schema Requirements

### Core Tables (8 tables)

#### 1. lessons
- `id SERIAL PRIMARY KEY`
- `title VARCHAR(255) NOT NULL`
- `content TEXT`
- `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- **Purpose:** Store lesson definitions and content

#### 2. concepts
- `id SERIAL PRIMARY KEY`
- `name VARCHAR(255) NOT NULL`
- `description TEXT`
- `mastery_threshold INTEGER DEFAULT 50`
- **Purpose:** Define learning concepts and mastery thresholds

#### 3. questions
- `id SERIAL PRIMARY KEY`
- `lesson_id INTEGER REFERENCES lessons(id) ON DELETE CASCADE`
- `text VARCHAR(1000) NOT NULL`
- `correct_answer TEXT`
- `concept_id INTEGER REFERENCES concepts(id)`
- `difficulty VARCHAR(20) DEFAULT 'medium'`
- `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- **Purpose:** Store quiz questions with relationships

#### 4. user_responses
- `id SERIAL PRIMARY KEY`
- `lesson_id INTEGER REFERENCES lessons(id) ON DELETE CASCADE`
- `question_id INTEGER REFERENCES questions(id) ON DELETE CASCADE`
- `user_answer TEXT`
- `is_correct BOOLEAN`
- `confidence INTEGER CHECK (confidence >= 1 AND confidence <= 5)`
- `response_time INTEGER`
- `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- **Purpose:** Track all user answers and confidence ratings

#### 5. learning_sessions
- `id SERIAL PRIMARY KEY`
- `user_id VARCHAR(255) NOT NULL`
- `lesson_id INTEGER REFERENCES lessons(id) ON DELETE SET NULL`
- `status VARCHAR(50) DEFAULT 'active'`
- `progress JSONB` - {totalQuestions, answered, currentQuestion, startTime}
- `mastery JSONB` - {concept_1: 75, concept_2: 40, ...}
- `started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- `completed_at TIMESTAMP`
- **Purpose:** Track active learning sessions with dynamic data

#### 6. voice_recordings
- `id SERIAL PRIMARY KEY`
- `session_id INTEGER REFERENCES learning_sessions(id) ON DELETE CASCADE`
- `question_id INTEGER REFERENCES questions(id) ON DELETE SET NULL`
- `audio_path VARCHAR(500)`
- `transcription TEXT`
- `confidence DECIMAL(5,4)`
- `duration INTEGER`
- `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- **Purpose:** Track voice quiz recordings and transcriptions

#### 7. voice_interactions
- `id SERIAL PRIMARY KEY`
- `session_id INTEGER REFERENCES learning_sessions(id) ON DELETE CASCADE`
- `recording_id INTEGER REFERENCES voice_recordings(id)`
- `action_type VARCHAR(50)` - playback, navigation, speed, volume, information, help
- `action_data JSONB` - {action, timestamp, metadata, ...}
- `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- **Purpose:** Track voice command executions and interactions

#### 8. audio_generations
- `id SERIAL PRIMARY KEY`
- `session_id INTEGER REFERENCES learning_sessions(id) ON DELETE CASCADE`
- `narration_text TEXT`
- `audio_path VARCHAR(500)`
- `duration INTEGER`
- `status VARCHAR(50)` - pending, processing, completed, failed
- `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- **Purpose:** Track TTS audio generation jobs and metadata

## Service Layer Requirements

### PostgreSQLDatabase Class Methods

#### Initialization
- `constructor(config)` - Initialize with database configuration
- `initialize()` - Test database connection
- `createTables()` - Create all tables if not exists
- `close()` - Close connection pool

#### Lesson Operations
- `createLesson(title, content)` - Insert new lesson
- `getLessonById(lessonId)` - Retrieve lesson by ID

#### Concept Operations  
- `createConcept(name, description, masteryThreshold)` - Create concept
- `getConceptById(conceptId)` - Retrieve concept by ID

#### Question Operations
- `createQuestion(lessonId, text, correctAnswer, conceptId, difficulty)` - Create question
- `getQuestionsByLesson(lessonId)` - Get all questions for lesson

#### Response Tracking
- `recordUserResponse(lessonId, questionId, userAnswer, isCorrect, confidence, responseTime)` - Record answer

#### Session Management
- `createLearningSession(userId, lessonId, progress, mastery)` - Start session
- `getLearningSession(sessionId)` - Retrieve session with parsed JSON
- `getActiveSession(userId, lessonId)` - Get user's active session
- `updateSessionProgress(sessionId, progress)` - Merge progress updates
- `updateSessionMastery(sessionId, mastery)` - Merge mastery updates
- `completeSession(sessionId)` - Mark session complete

#### Analytics
- `getWeakAreas(sessionId, threshold)` - Identify concepts < threshold
- `generateRecommendations(sessionId)` - Priority-based recommendations
- `getLearningStatistics(userId)` - User activity summary

#### Voice Operations
- `recordVoiceRecording(sessionId, questionId, audioPath, transcription, confidence, duration)`
- `recordVoiceInteraction(sessionId, recordingId, actionType, actionData)`
- `recordAudioGeneration(sessionId, narrationText, audioPath, duration, status)`

## Constraint Requirements

### Foreign Key Constraints
- **CASCADE**: Delete dependent records when parent deleted
- **SET NULL**: Allow nullable references when parent deleted
- All relationships must be properly enforced

### Check Constraints
- `confidence`: INTEGER, 1-5 range
- `duration`: INTEGER, > 0
- `status`: VARCHAR in allowed set (active, completed, pending, processing, failed)

### Not Null Constraints
- Required fields marked NOT NULL
- Automatic defaults for timestamps and IDs

### Indexes (Performance)
- `idx_user_responses_session` - lesson_id
- `idx_learning_sessions_user` - user_id
- `idx_learning_sessions_status` - status

## Error Handling Requirements

### Database Errors
- Connection failures: Detect and report immediately
- Query errors: Rollback transactions, return descriptive errors
- Transaction failures: Automatic rollback on error

### Business Logic Errors
- Invalid session IDs: Return success=false with error message
- Missing required fields: Fail fast with clear error
- Constraint violations: Handle gracefully with error details
- Foreign key violations: Detect and report to caller

### Error Response Format
```json
{
  "success": false,
  "error": "Descriptive error message",
  "code": "ERROR_CODE"
}
```
