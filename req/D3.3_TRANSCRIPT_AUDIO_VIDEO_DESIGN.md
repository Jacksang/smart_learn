# D3.3 Transcript/Audio/Video Support - Design Document

## Purpose
This document describes the design for comprehensive transcript, audio, and video file parsing support in Smart Learn.

## Overview
Smart Learn needs to support multiple file formats for content ingestion beyond basic text and PDF files. This includes:
- Subtitle files (.srt, .vtt) for video transcripts
- Audio files (mp3, wav, m4a, flac) for voice content
- Video files (mp4, avi, mov, mkv) for video-based learning

## Supported Formats

### Subtitle Files
- **SRT (SubRip Subtitle)**: `.srt`
  - Format: `HH:MM:SS,mmm --> HH:MM:SS,mmm`
  - Contains timestamp blocks and text content
- **VTT (WebVTT)**: `.vtt`
  - Format: `HH:MM:SS.mmm --> HH:MM:SS.mmm`
  - Web standard for web-based subtitles

### Audio Files
- **MP3**: `.mp3` - Most common audio format
- **WAV**: `.wav` - Uncompressed audio
- **M4A**: `.m4a` - Apple Audio format
- **FLAC**: `.flac` - Lossless audio

### Video Files
- **MP4**: `.mp4` - Most common video format
- **AVI**: `.avi` - Audio Video Interleave
- **MOV**: `.mov` - Apple QuickTime
- **MKV**: `.mkv` - Matroska Video

## Architecture

### File Ingestion Service
```javascript
backend/src/ingestion/
├── transcript.js      # Subtitle file parsing (COMPLETE)
├── audio.js          # Audio transcription service (IN PROGRESS)
├── video.js          # Video frame extraction (PLANNED)
└── service.js        # Routing and file type detection
```

### Processing Pipeline
1. **File Upload** → Store temporarily in `/tmp/`
2. **Type Detection** → Auto-detect file extension
3. **Parser Selection** → Route to appropriate parser
4. **Content Extraction** → Extract text, timestamps, metadata
5. **Database Storage** → Save to PostgreSQL with material metadata

### Parser Interface
```javascript
async function parse(filePath) {
  return {
    text: string,           // Extracted content
    timestamps: [],         // Optional timing information
    metadata: {             // File metadata
      fileName: string,
      fileType: string,
      fileSize: number,
      duration: string,     // For audio/video
      processedAt: string
    }
  };
}
```

## Dependencies

### External Tools
- **Whisper CLI**: For audio transcription
  - Install: `pip install whisper` or use pre-built binaries
  - Model: base, small, medium (configurable)
  
- **FFmpeg**: For video processing
  - Install: `apt-get install ffmpeg` or `brew install ffmpeg`
  - Used for frame extraction and metadata

### Node.js Libraries
- **fs**: File system operations (built-in)
- **child_process**: Execute CLI tools (built-in)

## Error Handling

### Parser Errors
- Unsupported file format → Throw `Error("Unsupported format: .xxx")`
- File not found → Throw `Error("File not found: path/to/file")`
- CLI tool missing → Throw `Error("CLI tool not found: whisper|ffmpeg")`

### Service Errors
- Temporary storage failure → Log and rollback
- Database save failure → Retry with exponential backoff

## Testing Strategy

## Security Considerations

### File Validation
- Check file extensions match content type
- Limit file sizes (max 500MB)
- Scan for malicious content

### Temporary Storage
- Clean up temporary files after processing
- Use isolated directories per upload
- Set expiration on temp files

## Performance Considerations

### Processing Time
- Subtitles: < 1 second for typical files
- Audio: Variable (depends on duration and model)
- Video: May take several minutes for long videos

### Asynchronous Processing
- Consider background job queue for large files
- Return job ID and status endpoint
- Notify user when processing complete
