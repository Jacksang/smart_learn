# D5.2 Concept Mapping - Design Document

## Purpose
This document describes the design for Smart Learn's Concept Mapping system - a visual organization tool that maps extracted concepts into interconnected graphs, linking them to lesson outline items for enhanced understanding and knowledge visualization.

## Overview
While the Flashcard Generation (D5.1) creates individual review items, Concept Mapping (D5.2) provides a holistic view of how concepts relate to each other and to the overall learning structure. This system:

1. **Analyzes** relationships between concepts (causal, hierarchical, associative)
2. **Visualizes** concepts in graph-based layouts (tree, network, timeline)
3. **Integrates** concepts with lesson outline items
4. **Highlights** coverage gaps and relationships for better understanding

This creates a powerful visualization tool that helps students see the "big picture" of their learning material.

## Core Principles

### 1. Relationship-First Design
- Concepts are organized by their relationships, not just keywords
- Multiple relationship types supported (causal, hierarchical, associative)
- Graph structure reveals hidden patterns and connections
- Centrality analysis identifies key concepts

### 2. Layout Flexibility
- Different layouts for different content types
- Tree layout for hierarchical knowledge (e.g., taxonomy)
- Network layout for associative knowledge (e.g., mind maps)
- Timeline layout for sequential knowledge (e.g., history)
- Automatic layout selection based on content analysis

### 3. Outline Integration
- Concepts linked to lesson outline items
- Visual indication of concept coverage
- Missing concepts identified and highlighted
- Progress tracking across outline structure

### 4. Interactive Exploration
- Zoom and pan capabilities for large concept maps
- Click-through to related flashcards
- Filter by concept type, coverage, or topic
- Export capabilities for sharing or printing

## Component 1: Concept Graph Analysis

### Graph Structure

**Nodes (Concepts):**
- `id`: Unique identifier
- `term`: Concept term
- `definition`: Concept definition
- `priority`: Importance score (0-1)
- `centrality`: Network centrality score (calculated)
- `type`: Concept type (term, definition, formula)
- `coverage`: Coverage status (covered, partial, missing)
- `outlineItemId`: Associated outline item (if any)

**Edges (Relationships):**
- `fromNodeId`: Source concept ID
- `toNodeId`: Target concept ID
- `type`: Relationship type (causal, hierarchical, associative)
- `strength`: Relationship strength (0-1)

**Graph Metadata:**
```json
{
  "nodes": [
    {
      "id": "concept_001",
      "term": "Photosynthesis",
      "definition": "Process by which plants...",
      "priority": 0.95,
      "centrality": 0.82,
      "type": "definition",
      "coverage": "covered",
      "outlineItemId": "outline_123"
    }
  ],
  "edges": [
    {
      "fromNodeId": "concept_001",
      "toNodeId": "concept_002",
      "type": "causal",
      "strength": 0.85
    }
  ],
  "metadata": {
    "totalNodes": 15,
    "totalEdges": 12,
    "density": 0.34,
    "avgCentrality": 0.45
  }
}
```

### Relationship Types

**1. Causal Relationships (A → B)**
- A causes B
- A leads to B
- A results in B
- A produces B
- **Strength:** Based on explicit language ("causes", "leads to", etc.)

**2. Hierarchical Relationships (A ⊂ B)**
- A is a type of B
- A is part of B
- A belongs to B
- A is classified as B
- **Strength:** Based on explicit classification language

**3. Associative Relationships (A ↔ B)**
- Related concepts that co-occur
- Frequently mentioned together
- Same broader category
- **Strength:** Based on co-occurrence frequency and context

### Centrality Analysis

**Degree Centrality:**
- Number of direct connections a concept has
- High degree = central concept
- Identifies key concepts in the network

**Betweenness Centrality:**
- Number of shortest paths passing through a concept
- High betweenness = bridge between clusters
- Identifies concepts that connect different knowledge areas

**Closeness Centrality:**
- Average distance from a concept to all others
- High closeness = can reach all concepts quickly
- Identifies concepts that provide efficient access to knowledge

**Eigenvector Centrality:**
- Importance based on connections to other important concepts
- Recursive importance calculation
- Identifies concepts that are connected to other key concepts

**Centrality Calculation:**
```javascript
function calculateCentrality(nodes, edges) {
  const centrality = {};
  
  // Degree centrality
  nodes.forEach(node => {
    centrality[node.id] = {
      degree: getDegree(node.id, edges),
      betweenness: calculateBetweenness(node.id, nodes, edges),
      closeness: calculateCloseness(node.id, nodes, edges),
      eigenvector: calculateEigenvector(nodes, edges)
    };
  });
  
  // Normalize scores to 0-1 range
  return normalizeCentrality(centrality);
}
```

## Component 2: Visual Layout Algorithms

### Layout Types

**1. Tree Layout**
- **Best for:** Hierarchical content
- **Structure:** Root concept at top, children below
- **Algorithm:** Depth-first traversal with level separation
- **Use cases:** Taxonomies, classification systems, process flows

**Layout Properties:**
```json
{
  "type": "tree",
  "rootNode": "concept_001",
  "levels": [
    ["concept_001"],
    ["concept_002", "concept_003"],
    ["concept_004", "concept_005", "concept_006"]
  ],
  "spacing": {
    "vertical": 100,
    "horizontal": 150
  }
}
```

**2. Network Layout**
- **Best for:** Associative content
- **Structure:** Force-directed graph layout
- **Algorithm:** Spring-force simulation
- **Use cases:** Mind maps, concept networks, knowledge graphs

**Layout Properties:**
```json
{
  "type": "network",
  "algorithm": "forceDirected",
  "parameters": {
    "springLength": 100,
    "springConstant": 0.01,
    "repulsion": 500,
    "iterations": 200
  },
  "positions": [
    {"id": "concept_001", "x": 320, "y": 240},
    {"id": "concept_002", "x": 450, "y": 200}
  ]
}
```

**3. Timeline Layout**
- **Best for:** Sequential/chronological content
- **Structure:** Linear timeline with events
- **Algorithm:** Temporal ordering with spacing
- **Use cases:** History, processes, development stages

**Layout Properties:**
```json
{
  "type": "timeline",
  "events": [
    {"id": "concept_001", "date": "1066-10-14", "x": 100},
    {"id": "concept_002", "date": "1215-06-15", "x": 200}
  ],
  "scale": {
    "start": "1066-01-01",
    "end": "2026-01-01",
    "pixelsPerYear": 2
  }
}
```

**4. Custom Layout**
- **Best for:** Mixed content types
- **Structure:** Hybrid of above layouts
- **Algorithm:** User-defined or AI-recommended
- **Use cases:** Complex topics with multiple knowledge types

### Layout Selection Algorithm

```javascript
function selectOptimalLayout(concepts, edges, contentMetadata) {
  const scores = {
    tree: 0,
    network: 0,
    timeline: 0,
    custom: 0
  };
  
  // Score tree layout
  const hierarchyScore = calculateHierarchyScore(edges);
  if (hierarchyScore > 0.7) {
    scores.tree = hierarchyScore * 1.2;
  }
  
  // Score network layout
  const densityScore = calculateNetworkDensity(edges);
  if (densityScore > 0.3) {
    scores.network = densityScore * 1.1;
  }
  
  // Score timeline layout
  const temporalScore = detectTemporalPatterns(concepts, edges);
  if (temporalScore > 0.6) {
    scores.timeline = temporalScore;
  }
  
  // Select highest score
  const bestLayout = Object.entries(scores)
    .sort((a, b) => b[1] - a[1])[0][0];
  
  return bestLayout;
}
```

## Component 3: Outline Integration

### Outline Concept Linking

**Integration Points:**
- Concepts linked to outline items by semantic matching
- Coverage status tracked (covered, partial, missing)
- Distribution analysis shows concept concentration
- Gap analysis identifies weak areas

**Linking Algorithm:**
```javascript
function linkConceptsToOutline(concepts, outlineItems) {
  const linked = [];
  
  concepts.forEach(concept => {
    const bestMatch = findBestOutlineMatch(concept, outlineItems);
    
    linked.push({
      concept: concept,
      outlineItem: bestMatch?.item || null,
      matchScore: bestMatch?.score || 0,
      isCovered: bestMatch?.score > 0.7,
      isPartial: bestMatch?.score > 0.4 && bestMatch?.score <= 0.7,
      isMissing: !bestMatch || bestMatch.score <= 0.4
    });
  });
  
  return linked;
}

function findBestOutlineMatch(concept, outlineItems) {
  let bestMatch = null;
  let bestScore = 0;
  
  outlineItems.forEach(item => {
    const score = calculateSemanticSimilarity(concept.term, item.text);
    if (score > bestScore) {
      bestScore = score;
      bestMatch = { item, score };
    }
  });
  
  return bestMatch;
}
```

### Coverage Analysis

**Coverage Metrics:**
- **Covered:** Concept fully mapped to outline item (match score > 0.7)
- **Partial:** Concept partially matched (match score 0.4-0.7)
- **Missing:** Concept not matched (match score ≤ 0.4)

**Distribution Analysis:**
- Concepts per outline item
- Coverage percentage per section
- Most/least covered topics
- Concept density by section

**Centrality Visualization:**

- Color intensity = priority score

## Component 4: Visualization Service Layer

### Service Functions

**1. Generate Concept Map**
- Extract concepts from materials
- Build concept graph
- Select optimal layout
- Calculate metrics (centrality, coverage)
- Return complete visualization data

**2. Get Concept Map**
- Retrieve existing map by ID
- Include linked flashcards
- Add coverage analysis
- Format for visualization

**3. Analyze Concept Map**
- Calculate network metrics
- Identify clusters and communities
- Find bridge concepts
- Generate insights

**4. Update Concept Map**
- Add new concepts
- Update relationships
- Recalculate metrics
- Refresh visualization

**Service Class:**
```javascript
class ConceptMappingService {
  constructor() {
    this.conceptGraph = new ConceptGraph();
    this.layoutAlgorithms = new LayoutAlgorithms();
    this.outlineIntegration = new OutlineIntegration();
  }
  
  async generateConceptMap(projectId, concepts, outlineItems = []) {
    // Build graph
    const graph = this.conceptGraph.build(concepts);
    
    // Calculate centrality
    this.conceptGraph.calculateCentrality(graph);
    
    // Select layout
    const layoutType = this.selectLayout(graph);
    const layout = this.layoutAlgorithms.generate(layoutType, graph);
    
    // Integrate with outline
    const linked = this.outlineIntegration.link(graph, outlineItems);
    
    return {
      graph,
      layout,
      coverage: this.analyzeCoverage(linked),
      insights: this.generateInsights(graph)
    };
  }
}
```

### Performance Optimization

**Caching Strategy:**
- Cache concept graphs by project + timestamp
- Cache layout calculations
- Cache coverage analysis
- Invalidate on content changes

**Optimization Techniques:**
- Lazy loading for large graphs
- Progress tracking for generation
- Incremental updates for modifications
- Batch processing for bulk operations

**Performance Targets:**
- Graph building: < 3 seconds for 50 concepts
- Layout generation: < 2 seconds
- Coverage analysis: < 1 second
- Complete map: < 6 seconds

### Database Schema (Recommended)
```sql
CREATE TABLE review_concept_maps (
  id UUID PRIMARY KEY,
  project_id UUID,
  title VARCHAR(255),
  layout_type VARCHAR(50),
  concepts_count INT,
  coverage_percentage DECIMAL(5,2),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_concepts (
  id UUID PRIMARY KEY,
  map_id UUID REFERENCES review_concept_maps(id),
  term VARCHAR(255),
  definition TEXT,
  centrality DECIMAL(3,2),
  priority DECIMAL(3,2),
  coverage_status VARCHAR(20),
  outline_item_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_relationships (
  id UUID PRIMARY KEY,
  map_id UUID REFERENCES review_concept_maps(id),
  from_concept_id UUID REFERENCES review_concepts(id),
  to_concept_id UUID REFERENCES review_concepts(id),
  relationship_type VARCHAR(50),
  strength DECIMAL(3,2),
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_concept_positions (
  map_id UUID REFERENCES review_concept_maps(id),
  concept_id UUID REFERENCES review_concepts(id),
  x DECIMAL(10,2),
  y DECIMAL(10,2),
  level INT,
  PRIMARY KEY (map_id, concept_id)
);
```
