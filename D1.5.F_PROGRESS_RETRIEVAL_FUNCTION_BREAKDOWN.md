## D1.5.F2 — Add repository helper for latest project snapshot retrieval
- [ ] Add repository method for latest project-level snapshot
- [ ] Enforce project ownership
- [ ] Add focused repository tests

### Business logic
A project owner should be able to request overall progress and get the newest project-level snapshot already persisted for that project.

### Exact coding problem
Implement one repository helper that returns the latest project-level `progress_snapshots` row for an owned project.

### Required function behavior
Suggested function name:
- `findLatestProjectSnapshotForUser({ projectId, userId })`

The helper must:
1. ensure the project belongs to the current user
2. fetch the newest project-level snapshot where `outline_item_id IS NULL`
3. return mapped fields suitable for API output
4. return `null` cleanly when no snapshot exists yet

### Required touch points
- `backend/src/progress/repository.js`
- `backend/src/progress/repository.test.js`

### Expected result
- a single ownership-safe repository method exists for latest project snapshot retrieval
- the returned shape is stable enough for controller use

### Expected test outcomes
- owned project returns latest project snapshot
- missing snapshot returns `null`
- cross-project/user leakage is blocked
