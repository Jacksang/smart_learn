## D1.3.A — Runtime and schema alignment audit for outlines
- [x] Inspect current PostgreSQL schema, repositories, routes, and controllers related to outlines
  - Inspected PostgreSQL schema in `backend/db/schema/001_baseline.sql`: `outlines(id, project_id, version_no, generation_source, status, summary, created_from_material_version, created_at, updated_at)` and `outline_items(id, outline_id, parent_item_id, title, summary, key_points, sort_order, depth_level, mastery_state, created_at, updated_at)` plus related indexes/triggers.
  - Inspected runtime outline code in `backend/src/outline/repository.js`, `backend/src/outline/controller.js`, `backend/src/outline/router.js`, `backend/src/outline/parser.js`, and route mount in `backend/config/server.js`.
  - Inspected adjacent PostgreSQL-style modules for comparison in `backend/src/projects/repository.js` and `backend/src/materials/repository.js`.
  - Inspected legacy data model in `backend/models/Outline.js` to confirm the older document-shaped outline contract still present in the repo.
- [x] Identify schema/runtime mismatches in the existing outline implementation
  - `backend/src/outline/repository.js` still targets a legacy document-style `outlines` shape (`user_id`, `course_title`, `subject`, `source_type`, `source_path`, `topics`, `ai_summary`) that does not exist in `backend/db/schema/001_baseline.sql`; the PostgreSQL schema expects `project_id`, `version_no`, `generation_source`, `status`, `summary`, and `created_from_material_version` instead.
  - Runtime list/create operations are scoped by `req.user.id` / `userId`, but the committed relational schema links outlines to `learning_projects.id`; there is no direct `outlines.user_id` column in the baseline schema.
  - The current implementation stores topic trees inline as a JSON `topics` column, while the committed schema normalizes topic hierarchy into `outline_items` rows (`outline_id`, `parent_item_id`, `title`, `summary`, `key_points`, `sort_order`, `depth_level`, `mastery_state`); `backend/src/outline/*` does not read or write `outline_items` at all.
  - `controller.js` and `router.js` expose only legacy list/create/upload flows for manual or file-backed outline blobs; there is no runtime support for outline versioning, `learning_projects.current_outline_id`, material-version linkage, or refresh/regeneration behavior implied by the schema.
  - `parser.js` emits legacy topic objects (`description`, `learningObjectives`, `difficulty`) shaped for the old Mongoose model in `backend/models/Outline.js`; those fields have no matching columns in `outline_items`, confirming the runtime contract is still aligned to the deprecated document model rather than the PostgreSQL MVP schema.
