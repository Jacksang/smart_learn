### D1.5.A audit notes (2026-04-05)
- Runtime gap found: `backend/src/answers/repository.js` still queries `answers` and legacy question fields (`topic`, `type`) even though `backend/db/schema/001_baseline.sql` defines `answer_attempts` and PostgreSQL-aligned `questions` fields.
- Runtime gap found: `backend/src/answers/controller.js` uses `findById(req.body.questionId)` and user-scoped attempt counting, but D1.5 routes in `docs/API_ENDPOINT_SPEC.md` are project-scoped and centered on `:projectId`, `:questionId`, and `answerAttemptId`.
- Runtime gap found: `backend/src/answers/router.js` exposes only top-level `GET /api/answers` and `POST /api/answers`, while the API spec defines:
  - `POST /api/projects/:projectId/questions/:questionId/answers`
  - `GET /api/projects/:projectId/questions/:questionId/answers`
  - `POST /api/projects/:projectId/answers/evaluate`
  - `GET /api/projects/:projectId/answers/history`
- Scope consequence: D1.5 should treat the current `backend/src/answers/*` module as a partial legacy stub and move it onto project ownership enforced through `learning_projects.user_id` and `questions.project_id`.
