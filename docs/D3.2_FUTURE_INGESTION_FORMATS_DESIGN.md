# D3.2 — Future Ingestion Formats (Design Doc)

## Purpose
Plan support for additional file formats beyond PDF/DOCX/images:
- Excel/CSV data ingestion
- PPT/slides ingestion
- Transcript (audio/video) ingestion

## Format Support Matrix

### Excel/CSV
**Use cases:**
- Structured data sets for learning
- Quiz banks in spreadsheet format
- Progress tracking exports
- Financial/e-commerce data samples

**Technical approach:**
- CSV: Native Node.js parsing (`csv-parse` or `papaparse`)
- Excel: `exceljs` or `xlsx` library for .xlsx/.xls
- Extract all sheets with metadata
- Handle formulas vs. raw values
- Support multi-sheet workbooks

**Storage strategy:**
- Flatten to text with sheet/range metadata
- Preserve column headers as context
- Store original file for reference

**Implementation priority:** Medium (useful for data-heavy courses)

---

### PPT/Slides
**Use cases:**
- Lecture slides as course material
- Presentation-based learning
- Visual content extraction

**Technical approach:**
- `pptx` library for .pptx files
- Extract text from slides
- Extract images with slide positioning
- Preserve slide order and transitions
- Handle embedded objects minimally

**Storage strategy:**
- Slide-by-slide text extraction
- Image extraction with slide reference
- Slide metadata (title, speaker notes if available)

**Implementation priority:** Medium-High (common educational format)

---

### Transcripts (Audio/Video)
**Use cases:**
- Lecture recordings
- Podcast-based courses
- Video tutorial transcripts
- Accessibility support

**Technical approach:**
- **Option A:** Integrate with OpenAI Whisper CLI (local, no API key)
  - Already installed in OpenClaw skill set
  - Use `openai-whisper` CLI for transcription
  - Process audio files locally
- **Option B:** Integrate with video-frames skill
  - Extract key frames from video
  - Combine with audio transcription
- **Option C:** Accept pre-transcribed text files (.srt, .vtt, .txt)

**Storage strategy:**
- Store transcript text with timestamps
- Link to original media file
- Extract and store thumbnail/frames if video
- Support subtitle file formats (.srt, .vtt)

**Implementation priority:** High (rich educational content)

## Storage Schema Considerations

### Current `source_materials` table
```sql
source_materials (
  id,
  project_id,
  file_name,
  file_type,  -- 'text', 'pdf', 'docx', 'image'
  file_path,
  extracted_text,
  weight,
  created_at,
  updated_at
)
```

### Potential Extensions

**Option 1: Extend `file_type` enum**
- Add: `'csv'`, `'excel'`, `'pptx'`, `'audio'`, `'video'`, `'subtitle'`

**Option 2: Add `content_metadata` JSONB column**
- Store format-specific metadata (sheet names, slide counts, transcript timestamps)
- Example:
  ```json
  {
    "sheets": ["Sheet1", "Sheet2"],
    "slides": 24,
    "duration_seconds": 3600,
    "transcript_segments": [...]
  }
  ```

**Recommended:** Option 2 (flexible, schema evolution friendly)
