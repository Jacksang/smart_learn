## User Flow

### First-Time Dashboard Visit
1. User opens progress dashboard
2. Loading skeleton displays
3. Data fetches from D7.1 APIs
4. Dashboard renders with:
   - Top metrics cards
   - Mastery breakdown sorted by priority
   - Progress trend chart
   - Weak areas highlighted
   - Personalized recommendations

### During Active Session
1. User answers questions in real-time
2. ProgressTracker records answer
3. Dashboard refreshes (every 5 min or on interaction)
4. Metrics update:
   - Accuracy recalculates
   - Response time updates
   - Mastery levels adjust
   - Recommendations may change

### Focused Study Session
1. User clicks on weak area
2. Dashboard navigates to practice
3. User completes practice questions
4. Dashboard updates with improved metrics
5. Next recommendation generated
