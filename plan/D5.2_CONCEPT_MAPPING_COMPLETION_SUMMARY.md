### 2. Visual Layout Algorithms ✅

**File:** `backend/src/review/layout-algorithms.js` (10,165 bytes)

**3 Layout Types:**

**1. Tree Layout**
- **Best for:** Hierarchical content (taxonomies, classifications)
- **Structure:** Root concept at top, children below
- **Algorithm:** Depth-first traversal with level separation
- **Use cases:** Biological classifications, process flows, category systems

**2. Network Layout**
- **Best for:** Associative content (mind maps, knowledge graphs)
- **Structure:** Force-directed graph simulation
- **Algorithm:** Spring-force simulation with repulsion/attraction
- **Use cases:** Concept networks, relationship maps, brainstorming

**3. Timeline Layout**
- **Best for:** Sequential/chronological content
- **Structure:** Linear timeline with temporal events
- **Algorithm:** Temporal ordering with spacing
- **Use cases:** History, processes, development stages

**Layout Selection Algorithm:**
```javascript
recommendLayout(graph) {
  // Score each layout type based on graph properties
  if (hierarchical edges > 50%):
    recommend: tree
  else if (density > 0.2):
    recommend: network
  else if (temporal content detected):
    recommend: timeline
  else:
    recommend: network (default)
}
```

**Layout Properties:**
```json
{
  "type": "network",
  "algorithm": "forceDirected",
  "positions": [
    {"id": "concept_001", "x": 320, "y": 240},
    {"id": "concept_002", "x": 450, "y": 200}
  ],
  "parameters": {
    "springLength": 100,
    "springConstant": 0.01,
    "repulsion": 500,
    "iterations": 200
  }
}
```

**Performance:**
- Force-directed simulation: ~200 iterations
- Layout generation: < 2 seconds for 50 nodes
- Centering normalization for better visualization

**Testing:** All layout types validated, algorithm selection accuracy, position calculations, performance benchmarks

```sql
CREATE TABLE review_concept_maps (
  id UUID PRIMARY KEY,
  project_id UUID,
  title VARCHAR(255),
  layout_type VARCHAR(50),
  concepts_count INT,
  coverage_percentage DECIMAL(5,2),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_concepts (
  id UUID PRIMARY KEY,
  map_id UUID REFERENCES review_concept_maps(id),
  term VARCHAR(255),
  definition TEXT,
  centrality DECIMAL(3,2),
  priority DECIMAL(3,2),
  coverage_status VARCHAR(20),
  outline_item_id UUID,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_relationships (
  id UUID PRIMARY KEY,
  map_id UUID REFERENCES review_concept_maps(id),
  from_concept_id UUID REFERENCES review_concepts(id),
  to_concept_id UUID REFERENCES review_concepts(id),
  relationship_type VARCHAR(50),
  strength DECIMAL(3,2),
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE review_concept_positions (
  map_id UUID REFERENCES review_concept_maps(id),
  concept_id UUID REFERENCES review_concepts(id),
  x DECIMAL(10,2),
  y DECIMAL(10,2),
  level INT,
  PRIMARY KEY (map_id, concept_id)
);
```
