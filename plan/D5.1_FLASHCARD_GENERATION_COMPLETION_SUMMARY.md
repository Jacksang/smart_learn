## Implementation Details

### 1. Concept Extraction System ✅

**File:** `backend/src/review/concept-extractor.js` (9,656 bytes)

**Key Features:**
- **Noun Phrase Identification:** Identifies technical terms and definitions
- **Definition Pattern Matching:** Recognizes "is defined as," "refers to," dash-separated, colon-separated patterns
- **Relationship Mapping:** Identifies causal (A causes B), hierarchical (A is a type of B) relationships
- **Concept Prioritization:** Multi-factor scoring (frequency, position, relationship density, definition quality)

**Extraction Algorithm:**
1. Preprocess content (normalize whitespace, remove boilerplate)
2. Identify noun phrases and technical terms
3. Extract definitions using pattern matching
4. Map relationships between concepts
5. Prioritize based on multiple scoring factors
6. Filter and format results

**Output Format:**
```json

    {
      "term": "Photosynthesis",
      "definition": "Process by which plants convert light energy...",
      "relatedTerms": ["chlorophyll", "glucose"],

      "confidence": 0.87
    }
  ],
  "metadata": {
    "totalExtracted": 15,
    "totalCandidate": 25,
    "processingTime": 1200
  }
}
```

**Testing:** Comprehensive pattern matching for all definition formats, frequency analysis, priority scoring validation

### 3. Spaced Repetition Algorithm ✅

**File:** `backend/src/review/spaced-repetition.js` (7,056 bytes)

**SM-2 Algorithm Implementation:**
- **Initial Interval:** 1 day
- **First Success:** 1 day interval
- **Second Success:** 6 day interval
- **Subsequent:** `newInterval = max(interval + 1, floor(interval * exp((rating - 3) * 0.135)))`

**Rating Scale (0-5):**
- **0-2:** Fail (reset to 1 day)
- **3:** Pass (minimal interval increase)
- **4:** Good (moderate increase)
- **5:** Easy (significant increase)

**Schedule Management:**
- Track repetitions per flashcard
- Calculate next review date dynamically
- Group by review dates for batch scheduling
- Handle time zone conversions
- Mastery detection (15+ reviews + 30+ day interval)

**Key Functions:**
- `initializeSchedule()` - Set up new flashcard
- `calculateNextReview()` - Update after review
- `getFlashcardsDue()` - Get due for review
- `getReviewSchedule()` - Get upcoming schedule
- `calculateReviewStats()` - Performance metrics
- `checkMastered()` - Mastery detection

**Schedule State:**

### 5. API Endpoints ✅

**Files:** 
- `backend/src/review/flashcard-controller.js` (5,678 bytes)
- `backend/src/review/flashcard-router.js` (1,537 bytes)

**6 API Endpoints:**

1. **POST /api/projects/:projectId/review/generate-flashcards**
   - Input: Materials array with content
   - Output: Generated flashcards with schedules
   - Use: Create new flashcards

2. **GET /api/projects/:projectId/review/flashcards/:flashcardId**
   - Input: Flashcard ID
   - Output: Flashcard with review stats
   - Use: View specific flashcard

3. **POST /api/projects/:projectId/review/review-flashcard**
   - Input: Flashcard ID, rating (0-5), optional responseTime
   - Output: Updated schedule
   - Use: Record review performance

4. **GET /api/projects/:projectId/review/review-schedule**
   - Input: Optional startDate, days (default 7)
   - Output: Review schedule for period
   - Use: View upcoming reviews

5. **GET /api/projects/:projectId/review/due**
   - Input: Optional targetDate
   - Output: Flashcards due for review
   - Use: Get today's review list

6. **GET /api/projects/:projectId/review/stats**
   - Input: None (projectId in URL)
   - Output: Student progress statistics
   - Use: View overall progress

**Error Handling:**
- Validated input parameters
- Clear error messages
- Appropriate HTTP status codes
- Consistent response format
