## 📊 API Endpoints to Implement

### **Password Reset (2 endpoints)**

#### 1. POST `/api/auth/password-reset/request`
- **Purpose:** Request password reset email
- **Input:** `{ email: string }`
- **Output:** `{ success: boolean, message: string }`
- **Validation:** Email format, user exists
- **Security:** Rate limit 5/hour per IP
- **Database:** Insert into `password_reset_tokens`

#### 2. POST `/api/auth/password-reset/complete`
- **Purpose:** Complete password reset with token
- **Input:** `{ token: string, newPassword: string }`
- **Output:** `{ success: boolean, message: string }`
- **Validation:** Token valid, password strength, token not expired (1 hour)
- **Security:** Delete used token, hash new password
- **Database:** Update `users.password_hash`, delete token

---

### **Email Verification (2 endpoints)**

#### 3. POST `/api/auth/verification/request`
- **Purpose:** Send verification email
- **Input:** `{ email: string }` (optional new email)
- **Output:** `{ success: boolean, message: string }`
- **Validation:** Email format, not already verified
- **Security:** Rate limit 3/hour per user
- **Database:** Insert into `email_verification_tokens`

#### 4. POST `/api/auth/verification/complete`
- **Purpose:** Verify email with token
- **Input:** `{ token: string }`
- **Output:** `{ success: boolean, verified_email: string }`
- **Validation:** Token valid, not expired (24 hours)
- **Security:** Delete used token, update user email
- **Database:** Update `users.email`, `users.email_verified`, delete token

---

### **Session Management (3 endpoints)**

#### 5. GET `/api/auth/sessions`
- **Purpose:** List active user sessions
- **Input:** None (protected)
- **Output:** `{ sessions: [{ id, device_info, ip_address, last_accessed_at, is_active }] }`
- **Validation:** User authenticated
- **Security:** Rate limit 60/minute
- **Database:** Query `user_sessions` for current user

#### 6. DELETE `/api/auth/sessions/:sessionId`
- **Purpose:** Revoke specific session
- **Input:** None (protected)
- **Output:** `{ success: boolean, message: string }`
- **Validation:** Session belongs to user, not current session
- **Security:** Soft delete (set `is_active = false`)
- **Database:** Update `user_sessions`

#### 7. DELETE `/api/auth/sessions/all`
- **Purpose:** Revoke all sessions (force logout)
- **Input:** None (protected)
- **Output:** `{ success: boolean, message: string }`
- **Validation:** User authenticated
- **Security:** Cannot revoke current session
- **Database:** Update `user_sessions` (set `is_active = false` for all except current)

---

### **OAuth Provider Linking (1 endpoint)**

#### 8. POST `/api/auth/oauth/link`
- **Purpose:** Link OAuth provider to user account
- **Input:** `{ provider: string, provider_user_id: string, access_token: string }`
- **Output:** `{ success: boolean, provider: string }`
- **Validation:** Provider supported, user authenticated
- **Security:** Verify OAuth token with provider
- **Database:** Insert into `oauth_users`

**Supported Providers:**
- Google
- Facebook
- GitHub
- Apple

## 🏗️ Database Schema

### **New Tables:**

#### 1. `password_reset_tokens`
```sql
CREATE TABLE password_reset_tokens (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    token VARCHAR(255) UNIQUE NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    used_at TIMESTAMP
);
```

#### 2. `email_verification_tokens`
```sql
CREATE TABLE email_verification_tokens (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    token VARCHAR(255) UNIQUE NOT NULL,
    email VARCHAR(255) NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    used_at TIMESTAMP
);
```

#### 3. `oauth_users`
```sql
CREATE TABLE oauth_users (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    provider VARCHAR(50) NOT NULL,
    provider_user_id VARCHAR(255) NOT NULL,
    access_token TEXT,
    refresh_token TEXT,
    token_expires_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(provider, provider_user_id)
);
```

### **Updated Tables:**

#### 4. `users` - Add columns:
- `email_verified` BOOLEAN DEFAULT false
- `last_password_reset_at` TIMESTAMP

## 🔐 Security Requirements

### **Password Reset:**
- Tokens expire after 1 hour
- Rate limit: 5 requests/hour per IP
- Secure token generation (cryptographically secure random)
- Email sent after token creation (async)

### **Email Verification:**
- Tokens expire after 24 hours
- Rate limit: 3 requests/hour per user
- Secure token generation
- Email sent after token creation (async)

### **Session Management:**
- Current session cannot be revoked
- Sessions expire after 7 days (automatic)
- Rate limit: 60 requests/minute for listing
- Rate limit: 10 requests/minute for revocation
- IP address binding (optional configuration)

### **OAuth Integration:**
- OAuth token verification with provider API
- Secure storage of access tokens (encrypted)
- Token refresh support (future)

### **JWT & Refresh Tokens:**
- Access tokens: 1 hour expiry, RS256 signing
- Refresh tokens: 30 days expiry, stored in database
- HTTP-only cookies for refresh tokens
- SameSite=Strict for CSRF protection
