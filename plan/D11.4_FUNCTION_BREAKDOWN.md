# D11.4 Notification System APIs — Function-Level Breakdown

**Date:** 2026-04-30
**Requirements:** `req/BACKEND_NOTIFICATION_SYSTEM_API_REQUIREMENTS.md`

## Task Slices

### Phase A: Database Migration
- [*] **A1** — Create `backend/migrations/20260430_add_notification_tables.sql`
  - CREATE TABLE notifications (id SERIAL PK, user_id FK→users, type, title, body, data JSONB, channel, priority, image_url, action_url, action_label, expires_at, sent_at, delivered_at, read_at, is_read, is_sent, is_delivered, failed_attempts, metadata JSONB, created_at, updated_at)
  - CREATE TABLE notification_templates (id SERIAL PK, name, type, subject, email_body, push_body, in_app_body, variables JSONB, is_active, created_at, updated_at)
  - CREATE TABLE notification_delivery_logs (id SERIAL PK, notification_id FK→notifications, channel, status, delivery_id, attempt, error_message, metadata JSONB, attempted_at)
  - Note: notification_preferences table already exists from D11.1
  - Indexes: idx_notifications_user, idx_notifications_type, idx_notifications_read, idx_notifications_created, idx_notifications_sent
  - Trigger for updated_at on notifications table

### Phase B: Repository Layer
- [ ] **B1** — Create `backend/src/notifications/repository.js`
  - `getNotifications(userId, {type, read, channel, page, limit})` — SELECT with filters, pagination
  - `getNotificationById(notificationId, userId)` — SELECT single notification
  - `getUnreadCount(userId)` — SELECT COUNT(*) WHERE is_read=false
  - `markAsRead(notificationId, userId)` — UPDATE is_read=true, read_at=NOW()
  - `markAllAsRead(userId)` — UPDATE all unread
  - `markBatchAsRead(userId, notificationIds)` — UPDATE batch
  - `deleteNotification(notificationId, userId)` — DELETE single
  - `deleteAllNotifications(userId, {type, read})` — DELETE with filters
  - `createNotification(data)` — INSERT new notification
  - `createDeliveryLog(notificationId, channel, status, deliveryId, errorMessage)` — INSERT log
  - `getPreferences(userId)` — SELECT from notification_preferences
  - `updatePreferences(userId, data)` — UPSERT notification_preferences
  - `getTemplate(name, type)` — SELECT from notification_templates

### Phase C: Service Layer
- [ ] **C1** — Create `backend/src/notifications/service.js`
  - `sendInApp(userId, type, title, body, data)` — Create notification in DB
  - `sendEmail(userId, type, templateData)` — Queue email via SendGrid (mock in MVP)
  - `sendPush(userId, type, templateData)` — Queue push via FCM (mock in MVP)
  - `sendMultiChannel(userId, type, title, body, data)` — Route to appropriate channels based on preferences
  - `shouldDeliver(userId, channel, type)` — Check preferences + quiet hours
  - `applyTemplate(templateName, variables)` — Resolve template with variables
  - `logDelivery(notificationId, channel, status, deliveryId, error)` — Record delivery attempt

### Phase D: Controller Layer
- [ ] **D1** — Create `backend/src/notifications/controller.js`
  - `getNotifications(req, res)` — GET /api/notifications
  - `getNotification(req, res)` — GET /api/notifications/:id
  - `markRead(req, res)` — POST /api/notifications/:id/read
  - `markAllRead(req, res)` — POST /api/notifications/mark-all-read
  - `markBatchRead(req, res)` — POST /api/notifications/read-all
  - `deleteNotification(req, res)` — DELETE /api/notifications/:id
  - `deleteAllNotifications(req, res)` — DELETE /api/notifications
  - `getPreferences(req, res)` — GET /api/notifications/preferences
  - `updatePreferences(req, res)` — PATCH /api/notifications/preferences
  - `sendTestNotification(req, res)` — POST /api/notifications/test
